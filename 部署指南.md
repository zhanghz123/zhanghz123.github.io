# GitHub Pages 部署指南

## 你的信息
- GitHub用户名：**zhanghz123**

## 两种部署方案

### 方案一：用户主页（推荐）

**访问地址：** `https://zhanghz123.github.io/`

**优点：** 简洁、专业，适合个人品牌

```bash
cd "D:/claude code/张老成的个人网站"

# 添加远程仓库（仓库名必须是 username.github.io）
git remote add origin https://github.com/zhanghz123/zhanghz123.github.io.git

# 重命名分支为 main
git branch -M main

# 推送到 GitHub
git push -u origin main
```

**然后：**
1. 在 GitHub 上创建仓库：`zhanghz123.github.io`（必须是这个名字）
2. 仓库设置为 Public
3. 推送后等待几分钟，访问 https://zhanghz123.github.io/

---

### 方案二：项目页面

**访问地址：** `https://zhanghz123.github.io/zhanglaocheng/`

**优点：** 可以创建多个项目网站

```bash
cd "D:/claude code/张老成的个人网站"

# 添加远程仓库
git remote add origin https://github.com/zhanghz123/zhanglaocheng.git

# 重命名分支为 main
git branch -M main

# 推送到 GitHub
git push -u origin main
```

**然后：**
1. 在 GitHub 上创建仓库：`zhanglaocheng`
2. 仓库设置为 Public
3. 推送后到仓库 Settings → Pages，选择 Source: Deploy from a branch, Branch: main/(root)
4. 等待几分钟，访问 https://zhanghz123.github.io/zhanglaocheng/

---

## 推荐使用方案一

用户主页更适合个人品牌展示，URL更简洁。

## 启用 GitHub Pages（方案一自动生效，方案二需要手动）

如果是方案二，推送后需要：

1. 打开 https://github.com/zhanghz123/zhanglaocheng/settings/pages
2. Source 选择：`Deploy from a branch`
3. Branch 选择：`main`，文件夹选择：`/(root)`
4. 点击 Save

## 自定义域名（可选）

如果想使用自己的域名，如 `www.zhanglaocheng.com`：

1. 在网站根目录创建 `CNAME` 文件
2. 文件内容填写：`www.zhanglaocheng.com`
3. 在域名服务商处添加 CNAME 记录指向 `zhanghz123.github.io`
